Published playbooks

Design asset skills for agents and creators.

These are the FreeDesignStore-native playbooks agents should follow before briefing, generating, uploading, reviewing, or publishing design assets. They are also exposed through MCP as list_design_skills, get_design_skill, and apply_design_skill.

before create

Asset Brief Builder

Turns a vague visual request into a concrete asset brief before creating or uploading.

  • Identify use case, asset type, size, and style.
  • Name rights and safety constraints.
  • Choose the right MCP creation path.
Open Markdown
vector art

SVG Illustration Director

Creates safe, reusable SVG illustrations ready for FDS catalog submission.

  • Define scene, style, palette, and accessibility text.
  • Avoid unsafe SVG markup.
  • Submit with complete catalog metadata.
Open Markdown
icons

Icon Set Designer

Designs coherent icons with consistent grid, stroke, naming, and export rules.

  • Use consistent viewBox and weight.
  • Prefer original symbols.
  • Avoid trademark confusion.
Open Markdown
photos

Stock Photo Curator

Chooses whether a photo should be FDS-hosted, uploaded by a rights holder, or linked off to Unsplash.

  • Check ownership and releases.
  • Use link-off for Unsplash.
  • Write useful category and tags.
Open Markdown
legal gate

License Safety Reviewer

Checks rights, attribution, unsafe markup, privacy, trademarks, and external-source handling.

  • Keep unclear rights pending.
  • Reject unsafe SVGs.
  • Separate FDS-hosted assets from linked sources.
Open Markdown
moderation

Publish Ready Asset Reviewer

Decides whether a pending design asset is ready to publish, needs revision, or should be rejected.

  • Review usefulness and metadata.
  • Check preview quality.
  • Publish only legal, clear assets.
Open Markdown

Connect via MCP

Claude Code, Codex, and any MCP client can connect in one command:

claude mcp add freedesignstore https://mcp.freedesignstore.online/mcp codex mcp add freedesignstore --url https://mcp.freedesignstore.online/mcp npx mcp-remote https://mcp.freedesignstore.online/mcp

Or add to your project's .mcp.json:

{ "mcpServers": { "freedesignstore": { "command": "npx", "args": ["mcp-remote", "https://mcp.freedesignstore.online/mcp"] } } }
